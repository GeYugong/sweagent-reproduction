# SWE-agent 论文精读笔记

## 文献信息

- 标题：SWE-agent: Agent-Computer Interfaces Enable Automated Software Engineering
- 作者：John Yang, Carlos E. Jimenez, Alexander Wettig, Kilian Lieret, Shunyu Yao, Karthik Narasimhan, Ofir Press
- 会议：NeurIPS 2024
- arXiv：2405.15793v3
- 论文文件：`paper/2405.15793_SWE-agent.pdf`
- 论文源码：`paper/2405.15793_source.tar.gz`

## 核心命题

论文把语言模型代理视为一种新的计算机终端用户。面向人类设计的 Shell、编辑器和搜索接口并不天然适合语言模型；动作空间、反馈格式、上下文维护和错误保护共同构成 Agent-Computer Interface（ACI），其设计可以在不修改模型权重的情况下显著改变任务表现。

ACI 同时定义：

- 代理可以执行的动作；
- 动作文档与调用格式；
- 环境状态和执行结果如何反馈；
- 历史观察如何保留、压缩和组合；
- 编辑或搜索失败后如何提供 guardrail。

## ACI 设计原则

1. 动作应简单，参数和文档保持紧凑。
2. 动作应高效，一次操作应完成一个有意义的高阶步骤。
3. 环境反馈应信息充分但避免无关上下文。
4. guardrail 应阻止错误扩散并缩短恢复路径。

## SWE-agent 的关键组件

- 搜索与导航：`find_file`、`search_file`、`search_dir`。
- 文件查看：围绕当前窗口显示固定行数，并维护位置状态。
- 文件编辑：单个命令执行多行修改，修改后立即展示更新窗口。
- lint/语法检查：阻止明显错误的编辑直接传播。
- 上下文管理：默认只保留最近五次观察，以控制历史噪声。
- 控制循环：每轮产生 thought 与 command，执行后把环境反馈送回模型。

## 主实验事实

论文使用的主要闭源模型为：

- `gpt-4-1106-preview`；
- `claude-3-opus-20240229`。

论文默认配置为：

- temperature：0.0；
- 文件窗口：100 行；
- 历史：最近 5 次观察；
- 带 demonstration；
- 带 linting 编辑器和摘要式搜索。

主结果：

| 系统 | SWE-bench Full | SWE-bench Lite | 平均成本（Full/Lite） |
|---|---:|---:|---:|
| RAG + GPT-4 Turbo | 1.31% | 2.67% | $0.13 / $0.13 |
| Shell-only + GPT-4 Turbo | 未报告 | 11.00% | 未报告 / $1.46 |
| SWE-agent + GPT-4 Turbo | 12.47% | 18.00% | $1.59 / $1.67 |
| SWE-agent + Claude 3 Opus | 10.46% | 13.00% | $2.59 / $2.18 |

GPT-4 Turbo 在完整 SWE-bench 上解决 286/2294 个任务，在 Lite 上解决 54/300 个任务。

## 核心消融

论文在 SWE-bench Lite 上报告的关键消融如下：

| 因素 | 默认 | 弱化设置 | 默认结果 | 弱化结果 |
|---|---|---|---:|---:|
| 编辑器 | 带 linting | 普通 edit | 18.0% | 15.0% |
| 编辑器 | 带 linting | 无专用 edit | 18.0% | 10.3% |
| 搜索 | 摘要式搜索 | 迭代式搜索 | 18.0% | 12.0% |
| 搜索 | 摘要式搜索 | 无专用搜索 | 18.0% | 15.7% |
| 文件窗口 | 100 行 | 30 行 | 18.0% | 14.3% |
| 文件窗口 | 100 行 | 全文件 | 18.0% | 12.7% |
| 上下文 | 最近 5 次观察 | 完整历史 | 18.0% | 15.0% |
| 上下文 | 最近 5 次观察 | 无 demonstration | 18.0% | 16.3% |

这些结果表明“提供更多信息”和“提供更接近人类工具的交互”并不必然更好。紧凑搜索、有限窗口和短历史共同减少了无效上下文与低效动作。

## 超参数搜索

附录在 37 个 SWE-bench dev 实例上进行搜索，每个配置报告 5 个样本的平均结果。GPT-4 Turbo 的最优组合是 temperature=0.0、window=100、Last5Observations，平均 resolve rate 为 15.1%。

该搜索集合很小，因此后续实验应把它视为工程选择证据，而不是稳定的总体最优证明。正式复现不能在 Lite test 上继续搜索这些参数。

## 复现威胁

1. `gpt-4-1106-preview` 已是历史模型标识，当前 API 是否仍可调用必须在实验时记录。
2. 论文没有在正文中给出唯一代码提交哈希。
3. 上游后续版本的 Lite 结果已变化，不能把新 README 的数字当作论文主表。
4. SWE-bench 的镜像、依赖源和仓库可用性会随时间变化。
5. temperature=0 并不保证远程模型或容器环境完全确定。
6. 论文结果的绝对复现可能受模型版本漂移影响，因此行为趋势和配对对照同样重要。
